```json

```

If consumed from another service, **prefer reusing that service's

## Idempotency strategy

Document explicitly how RULE-009/RULE-010 are honored: which columns
(typically `causation_id`) are queried before doing work, and what the
NO-OP path is.

## Definition of Done (Data model)

- [ ] Every aggregate has at least one invariant declared.
- [ ] `version` field present on every aggregate that is updated
      (RULE-008 — optimistic locking).
- [ ] Domain events are pure POJOs (no Spring, no Avro).
- [ ] Outbox payload distinct from domain event when they cross a
      service boundary.
- [ ] Every Kafka topic has an Avro schema with explicit compatibility mode.
- [ ] Idempotency strategy stated for every consumer-side handler.
